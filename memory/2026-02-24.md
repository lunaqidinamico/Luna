- **Lições Aprendidas sobre `exec` Elevado e `approval-timeout`:**
    - O comando `/elevated full` no chat é crucial para permitir que o agente execute comandos elevados `$exec(elevated=True)` sem a necessidade de aprovação explícita e em tempo real para cada execução, contornando o loop de `approval-timeout`.
    - Isso confirmou que a configuração `tools.elevated.allowFrom.webchat: true` no `openclaw.json` é uma permissão de *capacidade*, mas a execução do comando `/elevated` na sessão é a *política de tempo de execução*.
- **Lições Aprendidas sobre Ambiente Python e `metaapi-cloud-sdk`:**
    - A instalação de pacotes Python deve sempre ser feita em **ambientes virtuais (`venv`)** para evitar conflitos com o sistema (`externally-managed-environment` erro).
    - Para criar `venv`, o pacote `python3-venv` é um pré-requisito (`sudo apt install python3.12-venv`).
    - A integração com `metaapi-cloud-sdk` em Python é altamente sensível à **sequência correta de inicialização e conexão**, e é propensa a `AttributeError` se os métodos forem chamados incorretamente.
    - O `MetaApi(TOKEN)` é um gerenciador de alto nível. O atributo `api.metatrader_account_api` é o cliente para as contas MT, e deste é que chamamos `get_account()`, que então retorna um objeto `account` que possui os métodos `connect()` e `wait_until_synchronized()`.
- **Lições Aprendidas sobre `invalid auth-token header` (Problema Persistente na MetaAPI):**
    - Esta é, de longe, o maior obstáculo. O erro `Failed to authorize user due to invalid auth-token header` persiste mesmo após múltiplas tentativas de correção do script e atualização do token.
    - Isso indica que o problema é na **validade ou nas permissões do token JWT fornecido pela MetaAPI**, e não na nossa lógica de uso do SDK.
    - É C-R-Í-T-I-C-O verificar no painel da MetaAPI se o token possui **todas as permissões necessárias para *todas as APIs relevantes*** (provisionamento, manager, streaming, etc.), e se ele está corretamente **associado à `ACCOUNT_ID` específica (`827528aa-5be8-4df7-9872-1ca5879ac247`)**.
    - Em caso de persistência, um contato com o suporte da MetaAPI seria o próximo passo lógico.
